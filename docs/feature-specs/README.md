# Feature Specifications

Create a feature specification before material reconstruction.

A specification should define:

- stable feature ID,
- user-visible purpose,
- canonical legacy behavior,
- required capabilities,
- optional capabilities,
- dependencies,
- settings and defaults,
- initialization behavior,
- enable/disable/dispose behavior,
- reload requirements,
- persistence behavior,
- accepted failure behavior,
- test acceptance criteria,
- owner and reviewer.

Feature specifications describe intended behavior. They must not silently encode an unproven implementation assumption as a requirement.
